# Directional Stimulus Prompting (定向刺激提示)

Directional Stimulus Prompting (DSP) 是一种较新的提示工程技术，旨在通过向大语言模型（LLM）提供特定的、离散的"线索"或"激励"（即 Directional Stimulus），来引导模型生成更符合预期的输出。

## 概念

传统的提示通常是直接给出指令和输入（例如："请总结这篇文章"）。

而 **Directional Stimulus Prompting** 在输入（Input）和输出（Output）之间引入了一个中间环节——**"定向刺激"（Directional Stimulus）**。

- **核心定义**：这些"刺激"可以是关键词、关键短语、摘要线索、特定的风格描述或思维路径提示。它们充当了"向导"的角色，告诉模型在处理输入时应该**关注什么**或**朝着哪个方向思考**。

- **形象比喻**：
  - **普通 Prompt**：就像老师给学生一个题目："写一篇关于春游的作文。"（学生可能写得很发散）。
  - **DSP**：老师给题目时附带了要求："写一篇关于春游的作文，**必须包含：放风筝、突然下雨、互帮互助**。"（这些关键词就是定向刺激，引导学生写出符合特定情节的作文）。

## 原理

### 基本机制

标准的语言模型预测概率是 $P(y|x)$，即给定输入 $x$ 生成 $y$。DSP 将其通过引入刺激 $z$ 变为 $P(y|x, z)$。这个 $z$（刺激）通常由一个更小的模型（策略模型）生成，或者由人工设计，用来弥补仅凭 $x$（原始输入）难以明确意图的缺陷。

**为什么能提升效果？**

1. **缩小解空间（约束引导）**：LLM 的知识库非常庞大，对于一个开放性问题，它有无数种回答方式。定向刺激就像一个漏斗，过滤掉不相关的路径，强制模型聚焦于特定的信息点。

2. **明确对齐意图**：用户有时很难在一个指令中把所有隐性要求说清楚。通过提供具体的"关键词"或"线索"，模型能更精准地理解用户的真实意图（例如，在摘要任务中，指定模型只关注"经济影响"而非"政治背景"）。

3. **弥补黑盒模型的不可控性**：当我们无法微调大模型（如 GPT-4）时，DSP 提供了一种通过优化"提示中的线索"来间接控制模型输出的方法。

## Prompt 模版

在实际应用中，DSP 通常采用` [原始输入] + [定向刺激/线索] + [指令]`的结构。

**通用模版结构：**

```
文章/输入: [此处放入原始长文本或数据]

提示词/关键词/指导线索: [此处放入定向刺激，如关键词列表、关键句、特定的侧重点]

指令: 基于上述文章和提供的线索，[此处放入具体任务指令，如生成摘要、重写等]。
```

### 举例

假设我们有一篇关于**新款智能手机评测**的长文章，我们希望模型**只针对"电池续航"和"充电"部分**生成摘要。

**你的输入 (Prompt)：**

```
文章:
最近发布的 Model X 手机配备了惊艳的 6.7 英寸 OLED 显示屏和最新的骁龙处理器。然而，讨论最多的是其 5000mAh 电池。在我们的测试中，这款手机连续视频播放时间超过 18 小时，比上一代提升了 20%。相机系统表现不错，但在低光环境下有些吃力。在充电方面，它支持 120W 快充，仅需 19 分钟即可从 0% 充至 100%，尽管充电头会有点热。软件体验流畅，但预装软件是个问题。

定向刺激 (线索):
关键词：5000mAh 电池，18 小时播放，120W 快充，19 分钟充满。

指令:
请根据提供的关键词作为指导，生成一篇专门关注该手机电源性能的文章摘要。
```

**AI 输出：**

```
Model X 手机在电源性能方面表现出色，配备了巨大的 5000mAh 电池。测试显示其连续视频播放续航能力令人印象深刻，超过 18 小时。此外，该设备支持 120W 快充技术，能够在大约 19 分钟内从 0% 充至 100%。
```

> **注意**：如果没有这些 Keywords 作为刺激，模型可能会浪费篇幅去描述屏幕和相机，导致电池部分的描述不够详细。

## 使用技巧

想要用好 Directional Stimulus Prompting，可以参考以下技巧：

1. **自动化生成刺激（进阶玩法）**：
   - 在高级应用（如学术论文中的方法）中，通常会训练一个特定的小模型（Policy Model，比如 T5）专门用来从长文中提取"关键词"或"线索"，然后把这些线索喂给大模型（如 GPT-4）。
   - **个人开发者做法**：你可以用一个 Prompt 先让 AI 提取关键点（Step 1），然后再用提取的关键点让 AI 写正文（Step 2）。

2. **关键词越具体越好**：
   - 差的刺激：关注优点。
   - 好的刺激：关键词：高投资回报率，无缝集成，7x24小时支持。

3. **适用于"长文摘要"和"风格迁移"**：
   - **摘要**：通过给出一组必须包含的实体名词，防止模型产生幻觉或遗漏关键数据。
   - **风格迁移**：将"幽默、讽刺、简短"作为刺激词，比单纯在指令里写"请用幽默的风格"效果往往更稳定。

4. **与 Chain of Thought (CoT) 的区别与结合**：
   - **CoT (思维链)** 侧重于**推理过程**（Step-by-step reasoning）。
   - **DSP (定向刺激)** 侧重于**关键信息点**（Key information extraction）。
   - **结合**：你可以先给模型"关键线索"（DSP），然后要求它"一步步根据线索推理"（CoT），效果通常最佳。

## 总结

Directional Stimulus Prompting 是一种"画重点"的技术。它不指望模型自己从一大堆信息中猜出你想看什么，而是显式地把"重点线索"提炼出来放在 Prompt 中。

- **核心公式**：`输入` + `具体线索 (刺激)` -> `更好的输出`

## 进阶应用模式

### 模式一：两阶段 DSP

```
阶段一（提取刺激）：
"请从以下文章中提取 5 个最重要的关键词：{文章}"

阶段二（生成内容）：
"基于文章和以下关键词：{提取的关键词}，生成一段摘要。"
```

### 模式二：多维度刺激

```
文章: {原文}

维度 1 - 关键实体: 公司A, 产品B, 事件C
维度 2 - 情感倾向: 积极, 谨慎乐观
维度 3 - 时间线索: 2024年Q1, 预期年底

指令: 请结合以上三个维度的线索，生成一篇客观中立的报道摘要。
```

### 模式三：对比刺激

```
文章 A: {原文 A}
文章 B: {原文 B}

对比线索:
- 相同点: 都提到了...
- 差异点: A 认为...而 B 认为...

指令: 请基于以上对比线索，生成一份对比分析报告。
```

## 与其他技术的结合

| 组合技术 | 组合方式 | 效果提升 |
|---------|---------|---------|
| **DSP + CoT** | 先用 DSP 提取线索，再用 CoT 推理 | 既聚焦又严谨 |
| **DSP + RAG** | 用 DSP 指导检索方向 | 提升检索相关性 |
| **DSP + Few-shot** | 在示例中展示如何响应刺激 | 示例驱动的效果增强 |
| **DSP + APE** | 用 APE 自动优化刺激词 | 自动化刺激生成 |

## 常见陷阱与规避

| 陷阱 | 表现 | 规避方法 |
|------|------|---------|
| **刺激过强** | 模型机械堆砌关键词，失去流畅性 | 平衡指令与刺激的比例 |
| **刺激冲突** | 多个刺激词互相矛盾 | 检查刺激词的一致性 |
| **刺激不足** | 刺激词过于笼统，无指导意义 | 使用具体、可验证的关键词 |
| **过度依赖** | 模型只关注刺激词，忽略其他重要信息 | 明确告知模型刺激是指导而非限制 |

> **实践建议**：DSP 是一种轻量级但效果显著的技术。在实际应用中，建议从手动定义刺激词开始，在积累足够数据后可考虑使用 APE 或训练小模型来自动化刺激生成。

# Multimodal CoT (多模态思维链)

Multimodal Chain-of-Thought (多模态思维链) 是传统文本思维链（CoT）的进化版。它允许模型同时利用**文本**和**视觉信息**（图像、图表）来生成中间推理步骤，从而解决那些单纯依靠文本无法处理的复杂问题。

## 概念

**Multimodal CoT** 的核心在于**跨模态推理**。

- **Traditional CoT (传统 CoT)**：仅基于文本输入进行逻辑推导。
  - _输入_："这只鸟是什么颜色的？"（若没有图片，无法回答）

- **Multimodal CoT (多模态 CoT)**：基于"图像 + 文本"输入，先提取视觉特征，再结合常识进行推导。
  - _输入_：`[图片]` + "这只鸟会飞吗？"
  - _推理过程_：
    1. **视觉识别**：图像中是一只企鹅。
    2. **常识检索**：企鹅属于鸟类，但翅膀退化，适应游泳。
    3. **逻辑判断**：虽然是鸟，但企鹅不会飞。
  - _输出_：不会。

## 原理

Multimodal CoT 将视觉编码器（Visual Encoder）与语言模型（LLM）结合，分两步工作：

1. **基本原理 (Rationale Generation)**：模型首先关注图像中的关键区域，并生成一段描述或分析（Rationale），解释它看到了什么以及这与问题有何关系。

2. **答案推断 (Answer Inference)**：模型结合原始问题和生成的 Rationale，推导出最终答案。

**为什么能提升效果？**

1. **视觉接地 (Visual Grounding)**：传统模型容易产生幻觉（Hallucination）。Multimodal CoT 强制推理过程"基于图片事实"，减少了凭空捏造。

2. **信息互补**：很多问题的线索隐藏在图片细节中（如几何题的辅助线、图表的数据趋势）。文本提供了逻辑框架，而图片提供了必要的数据支撑，两者缺一不可。

3. **模拟人类感知**：人类在解决物理世界的问题时，本就是"看图 → 思考 → 结论"。Multimodal CoT 更加符合人类的认知直觉。

## Prompt 模版

要写出高质量的多模态 CoT 提示词，建议遵循 **"观察 - 关联 - 推理"** 的三段式结构。直接把图片扔给 AI 有时会遗漏细节，通过 Prompt 强制它"看清楚"是关键。

**通用 Prompt 结构：**

```
[插入图片]
角色：你是一名[专家角色，如：建筑工程师 / 数据分析师]。
任务：请基于这张图片回答问题：[你的具体问题]。

请按照以下三个步骤进行思考：
1. 视觉观察 (Observation)：首先，详细描述你在图片中看到的[关键细节/数据趋势/物体位置]。不要遗漏任何微小的线索。
2. 逻辑连接 (Reasoning)：结合上述观察和你的专业知识，一步步分析这些细节意味着什么。
3. 最终结论 (Conclusion)：给出明确的答案。
```

### 举例

假设我们有一道初中物理题，配有一张电路图。我们**严格套用上述模版**进行提问，以确保推理的准确性。

**你的输入：**

```
[图片：一个包含电池、开关（断开状态）和两个并联灯泡的电路图]

角色：你是一名物理老师。
任务：请基于这张图片回答问题：闭合开关后，如果上方灯泡的灯丝断了，下方灯泡会亮吗？

请按照以下三个步骤进行思考：
1. 视觉观察 (Observation)：首先，详细描述你在图片中看到的电路连接方式。不要遗漏任何微小的线索。
2. 逻辑连接 (Reasoning)：结合上述观察和你的专业知识，一步步分析故障对电路的影响。
3. 最终结论 (Conclusion)：给出明确的答案。
```

**AI 输出：**

```
好的，老师来一步步分析这个问题：

1.  **视觉观察 (Observation)**：
    * 我看到一个包含电池组、一个开关和两个灯泡的电路。
    * **连接方式**：这是一个典型的**并联电路**。电流从电池正极出发后，分成了两条路径（支路），分别流向两个灯泡，最后汇合流回负极。
    * **开关位置**：开关位于干路上（靠近电池的位置），这意味着它控制着整个电路的通断。

2.  **逻辑连接 (Reasoning)**：
    * **初始状态**：题目条件是"闭合开关"，这意味着干路导通，电流可以流出电池。
    * **故障分析**："上方灯泡灯丝断了"，在物理学中这意味着上方的那条支路发生了**断路 (Open Circuit)**，电流无法通过上方支路。
    * **并联特性**：根据并联电路的特性，**各支路互不影响**。上方支路虽然断了，但这并不影响下方支路与电源构成的回路。电流依然可以通过下方支路流回电池负极。

3.  **最终结论 (Conclusion)**：
    * 基于上述分析，下方支路的回路是完整的，电流可以畅通无阻。
    * **答案**：会亮。
```

> **注意**：如果是纯文本模型，可能无法理解"上方"和"下方"分别指代电路图中的哪一部分，从而导致推理混乱。

### 使用技巧

1. **显式引导视觉关注**：在 Prompt 中加入"请仔细观察图片的`[某部分]`"或"请描述图中的`[关键物体]`"。
   - _示例_："请先描述图表中的红色折线趋势，再预测明年的数据。"

2. **两阶段提示 (Two-stage Prompting)**：
   - **阶段一**：让 AI 生成图片描述（Image Caption）或提取文字（OCR）。
   - **阶段二**：将生成的描述作为上下文，输入给 LLM 进行 CoT 推理。
   - _适用场景_：当你的模型多模态能力较弱，但文本推理能力很强时。

3. **处理图表与几何题**：对于包含数据的图片，要求 AI "先将图表转换为数据表格"，然后再进行计算。这能显著提高数值计算的准确率。

## 额外补充：应用场景对比

| 场景 | 传统文本 CoT | 多模态 CoT |
|------|-------------|-----------|
| **数学应用题** | 适用（纯文字描述） | **适用（含几何图形、函数图像）** |
| **文档理解** | 适用（纯文本 PDF） | **适用（含表格、扫描件、手写体）** |
| **科学问答** | 适用（基于知识库） | **适用（基于实验现象图、生物结构图）** |
| **导航/机器人** | 不适用 | **核心能力（基于摄像头画面决策）** |
| **医疗影像分析** | 不适用 | **核心能力（基于 X 光/CT 影像诊断）** |

## 模型能力要求

并非所有模型都支持 Multimodal CoT：

| 模型类型 | 视觉理解 | 适用性 |
|---------|---------|--------|
| GPT-4V / Claude 3 | ✅ 原生支持 | 最佳 |
| Gemini Pro Vision | ✅ 原生支持 | 优秀 |
| 纯文本 LLM | ❌ 不支持 | 需先用其他工具提取图像描述 |

## 局限性与应对策略

### 局限性
1. **模型依赖**：需要多模态大模型支持，成本高于纯文本模型
2. **细节遗漏**：模型可能忽略图片中的细微之处（如小字标注、隐藏线条）
3. **空间理解**：对复杂三维空间关系的理解仍有局限

### 应对策略
- **分块处理**：将大图切分，逐一分析后再综合
- **人工标注**：在关键区域添加箭头或文字标注，引导模型关注
- **多轮确认**：针对关键细节进行追问验证

> **实践建议**：对于关键业务场景（如医疗诊断），Multimodal CoT 应作为辅助工具，最终决策需人工审核。

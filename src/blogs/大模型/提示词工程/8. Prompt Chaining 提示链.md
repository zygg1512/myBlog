# 提示链 (Prompt Chaining)

提示链（Prompt Chaining）是一种进阶的提示工程（Prompt Engineering）技术。它通过将一个庞大、复杂的任务拆解为一系列相互关联的小任务，引导 AI 分步骤完成，从而获得比单次提示更精准、更可控的结果。

## 概念

**Prompt Chaining（提示链）** 指的是将大语言模型（LLM）的输出作为下一个提示的输入，从而形成一个链条式的任务处理流程。

这就好比工厂的流水线：

- **单次提示 (Single Prompt)**：试图让一个工人在一个工作台上同时完成零件筛选、组装、喷漆和打包。容易出错，顾此失彼。

- **提示链 (Prompt Chaining)**：
  - 工人 A 负责筛选（Prompt 1） → 产出物交给工人 B。
  - 工人 B 负责组装（Prompt 2） → 产出物交给工人 C。
  - 工人 C 负责喷漆（Prompt 3）。

在对话框中，表现为多轮对话；在程序开发中，表现为将上一次 API 调用的 output 插入到下一次 input 中。

## 原理

为什么拆分任务比"写一个超级详细的长提示"效果更好？主要基于以下原理：

1. **降低认知负荷 (Cognitive Load Reduction)**：LLM 的注意力机制（Attention Mechanism）是有限的。当提示词过长、指令过于复杂（既要翻译，又要摘要，还要改写风格，还要提取关键词）时，模型容易产生幻觉或忽略部分指令。拆分任务让模型每次只专注于一件事。

2. **上下文清洗与纠错 (Context Cleansing & Correction)**：在链条的中间环节，人类（或代码逻辑）可以检查中间结果。如果第一步错了，可以在第二步开始前修正，避免错误累积。

3. **利用中间思维 (Intermediate Reasoning)**：很多复杂任务无法直接得出答案。通过强制模型先输出"思考过程"或"结构化数据"，后续步骤基于这些高质量的中间产物生成最终结果，质量会大幅提升。

4. **突破 Token 限制**：对于极长的文档处理，单次输入可能超出上下文窗口。提示链可以分段处理再汇总。

## 举例

假设我们需要完成一个任务：**从一篇冗长杂乱的会议记录中，起草一封发给客户的专业项目进度邮件。**

如果直接让 AI 写，它可能会遗漏细节或语气不当。我们使用 Prompt Chaining 将其拆分为三步。

### 第一步：提取关键信息（清洗数据）

你的输入：

```
请分析以下混乱的会议记录文本，提取出：1. 已完成的里程碑；2. 遇到的阻碍；3. 下一步计划。请用简洁的列表格式输出，不要废话。

[会议记录文本]：
大家早上好。关于 Alpha 项目，那个后端 API 接口基本搞定了，测试通过。但是前端页面现在卡在设计图确认上，客户还没回复。哦对了，下周我们要开始做数据库迁移。之前说的服务器扩容已经完成了。就是这样。
```

AI 输出：

```
1. **已完成的里程碑**：
   - 后端 API 接口开发完成并通过测试。
   - 服务器扩容完成。

2. **遇到的阻碍**：
   - 前端页面开发受阻，原因是客户尚未确认设计图。

3. **下一步计划**：
   - 下周开始进行数据库迁移。
```

### 第二步：风格转化与撰写（生成草稿）

你的输入：

```
基于你刚才提取的以下关键信息，起草一封发给客户（张总）的邮件。
要求：
- 语气专业、客气但坚定。
- 重点强调我们已完成的工作。
- 委婉地催促客户确认设计图，说明这会影响进度。

[关键信息]：
1. 已完成：后端 API 接口、服务器扩容。
2. 阻碍：前端页面待客户确认设计图。
3. 计划：下周数据库迁移。
```

AI 输出：

```
主题：Alpha 项目进度汇报及设计图确认事宜

尊敬的张总：

您好！

特此向您汇报 Alpha 项目的最新进展。经过团队的努力，我们已经顺利完成了以下关键里程碑：
1. 后端 API 接口已开发完毕并通过测试；
2. 服务器扩容工作已全部完成。

目前项目整体推进顺利，但为了确保前端开发工作能按时衔接，我们需要尽快确定设计图方案。建议您在百忙之中抽空确认，以便我们团队能即刻开展页面开发，避免影响后续进度。

此外，我们计划于下周启动数据库迁移工作。

期待您的回复。

祝好，

项目团队
```

### 第三步：格式化与润色（最终输出）

你的输入：

```
请将上面的邮件内容转换为 HTML 格式，以便我直接复制到邮件营销工具中。同时，将"祝好"改为"顺颂商祺"。
```

AI 输出：

```

    <p>尊敬的张总：</p>
```

## 使用技巧

1. **定义清晰的"交接棒"格式**：在链条的中间环节，要求 AI 输出结构化的数据（如 JSON、Markdown 列表、XML）。这比纯文本更容易被下一环节的 Prompt 准确读取和处理。
   - _坏例子_："写个总结。"
   - _好例子_："请以 JSON 格式输出总结，包含 key: 'summary' 和 'keywords'。"

2. **设置人工检查点 (Human-in-the-loop)**：不要盲目地自动执行整个链条。在关键步骤（如生成最终文案前）检查中间输出。如果第一步提取的信息是错的，后面写得再好也是废的。

3. **模块化调试**：如果你发现最终结果不好，不要只改最后一步的提示词。去检查是哪一步"掉链子"了。通常问题出在前面的步骤信息丢失或理解偏差。

4. **利用"定界符"**：在将上一步的输出作为下一步的输入时，使用 `"""` 或 `###` 等符号将上下文括起来，告诉 AI："这是背景资料，不是我对你的指令。"

## 提示链 vs 思维链 (CoT)

很多初学者容易混淆 **Prompt Chaining** 和 **Chain of Thought (CoT)**，虽然它们都旨在解决复杂问题，但有本质区别：

| 维度 | 思维链 (Chain of Thought) | 提示链 (Prompt Chaining) |
|------|--------------------------|-------------------------|
| **交互次数** | **单次**交互。 | **多次**交互（或多次 API 调用）。 |
| **核心指令** | "Let's think step by step" (请一步步思考)。 | Step 1 做 X -> Step 2 做 Y -> Step 3 做 Z。 |
| **可见性** | 思考过程在一个回复中完成，内部逻辑连续。 | 思考过程被拆分为显式的多个步骤，中间产物可见。 |
| **适用场景** | 数学推理、逻辑谜题、需要自我修正的短任务。 | 文档编写、跨语言处理、需要引用外部工具、极长流程的任务。 |
| **错误处理** | 单次机会，错误会直接影响结果 | 可插入检查点，错误可在下游修正 |
| **实现复杂度** | 低（单次调用） | 中（需要编排逻辑） |

**总结：** 如果你需要 AI 给你一个结果，但过程很绕，用 **CoT**。如果你需要 AI 帮你完成一个工作流，且每个步骤的产出都很重要，用 **Prompt Chaining**。

## 实现架构示例

```python
# 伪代码示例
def prompt_chain_workflow(raw_input):
    # Step 1: 提取关键信息
    extracted = llm.generate(
        prompt=f"从以下文本提取关键信息：{raw_input}",
        output_format="json"
    )

    # 人工检查点（可选）
    if not validate_extraction(extracted):
        return "提取失败，需要人工介入"

    # Step 2: 生成内容
    draft = llm.generate(
        prompt=f"基于以下信息生成邮件：{extracted}"
    )

    # Step 3: 格式化输出
    final = llm.generate(
        prompt=f"将以下内容转为HTML：{draft}"
    )

    return final
```

## 常见模式与变体

| 模式 | 描述 | 适用场景 |
|------|------|---------|
| **线性链** | A → B → C，单向流动 | 标准流水线处理 |
| **分支链** | A → (B1, B2)，然后合并 | 并行处理不同方面 |
| **循环链** | A → B → 检查 → 可能回到 A | 质量不达标时重做 |
| **条件链** | 根据中间结果选择不同分支 | 需要动态决策的场景 |

## 最佳实践

1. **保持步骤精简**：每个 Prompt 只做一件事，步骤过多会增加累积错误风险
2. **版本控制**：对每个步骤的 Prompt 进行版本管理，便于回溯优化
3. **监控与日志**：记录每个步骤的输入输出，便于问题排查
4. **优雅降级**：设计备选路径，当某步骤失败时提供默认处理

> **实践建议**：Prompt Chaining 是构建复杂 AI 应用的基石技术。在评估平台、自动化报告生成、多语言翻译流水线等场景中，它能显著提升输出质量和系统可控性。

# Active-Prompt (主动提示)

Active-Prompt 是一种旨在优化大型语言模型（LLM）上下文学习（In-Context Learning）效果的高级策略。它由 Shizhe Diao 等人在 2023 年提出，旨在解决思维链（Chain-of-Thought, CoT）中示例选择的痛点。

## 概念

**Active-Prompt** 的核心是将机器学习中的 **"主动学习" (Active Learning)** 思想引入到 Prompt 工程中。

在传统的 Few-Shot（少样本）提示中，我们通常随机选择示例或人工挑选"典型"示例给模型看。但 Active-Prompt 认为：**与其随机给例子，不如找出模型最"困惑"或最"不确定"的问题，让人类专家针对这些特定问题编写详细的思维链（CoT），再作为示例喂给模型。**

简单来说，它的流程是：

1. **测试：** 让模型回答一系列问题。
2. **侦测：** 找出模型回答得不确定、动摇或错误的问题。
3. **教学：** 人类针对这些"难点"写出正确的推理步骤。
4. **应用：** 将这些经过人工修正的难点作为 Prompt 中的示例，去解决其他问题。

## 原理

### 核心机制：不确定性度量 (Uncertainty Estimation)

Active-Prompt 的有效性建立在解决**信息量最大**的样本上。

1. **弥补认知短板**：如果模型对某个问题非常确定（比如 1+1=2），将这个问题作为示例放入 Prompt 中，对模型的启发很小（信息增益低）。相反，模型感到"困惑"的问题，往往包含了复杂的逻辑陷阱或模型尚未掌握的推理模式。

2. **从错误中学习**：通过度量不确定性（例如：让模型回答同一个问题 5 次，如果 5 次答案各不相同，说明不确定性极高），我们可以精准定位模型的弱点。

3. **针对性优化**：针对这些高不确定性的问题编写 CoT，相当于给模型做了"错题集"强化训练。模型学会了最难的逻辑后，处理简单或中等难度的问题就会游刃有余。

## 举例

假设我们要解决一个复杂的**逻辑数学问题**。

### 第一阶段：探测不确定性 (User 操作)

首先，你有一批训练题目，但不知道选哪道题作为 Prompt 的示例最好。你将其中一道题丢给 AI **跑 3-5 次**。

**你的输入 (测试题)：**

```
问题：一个箱子里有 5 个红球，3 个蓝球。拿走 1 个红球后，放入 2 个绿球。现在箱子里有多少个球？
请回答。
```

**AI 输出 (多次尝试的结果)：**

- 第一次：9 个 (5-1+2+3) → 正确
- 第二次：7 个 (忽略了蓝球) → 错误
- 第三次：8 个 (计算错误) → 错误

**结论：** 这道题 AI 每次回答都不一样（不一致性高），说明这是一道**高不确定性**的题目，非常有价值，**选中它！**

### 第二阶段：人工编写思维链 (Human 操作)

你针对这道题，写出完美的推理步骤。

**人工编写的 CoT：**

> 让我们一步步思考。
>
> 1. 起始状态：5 个红球，3 个蓝球。总数 = 5 + 3 = 8。
> 2. 操作一：拿走 1 个红球。红球剩余 4 个。总数 = 8 - 1 = 7。
> 3. 操作二：放入 2 个绿球。总数 = 7 + 2 = 9。
> 4. 最终状态：4 红，3 蓝，2 绿。 因此答案是 9 个。

### 第三阶段：构建最终 Active-Prompt (应用)

**你的输入 (最终 Prompt)：**

```
你是一个数学推理专家。请参考以下示例的逻辑进行回答。

问题：一个箱子里有 5 个红球，3 个蓝球。拿走 1 个红球后，放入 2 个绿球。现在箱子里有多少个球？
答案：让我们一步步思考。
1. 起始状态：5 个红球，3 个蓝球。总数 = 5 + 3 = 8。
2. 操作一：拿走 1 个红球。红球剩余 4 个。总数 = 8 - 1 = 7。
3. 操作二：放入 2 个绿球。总数 = 7 + 2 = 9。
4. 最终状态：4 红，3 蓝，2 绿。
因此答案是 9 个。

问题：小明有 10 块糖，吃了一半，又像妈妈要了 3 块，最后分给妹妹 2 块。小明现在有几块？
答案：
```

**AI 输出 (提升后的效果)：**

```
让我们一步步思考。
1. 起始状态：小明有 10 块糖。
2. 操作一：吃了一半。10 的一半是 5，剩下 10 - 5 = 5 块。
3. 操作二：向妈妈要了 3 块。现在有 5 + 3 = 8 块。
4. 操作三：分给妹妹 2 块。现在有 8 - 2 = 6 块。
因此答案是 6 块。
```

> **注意**：如果使用普通 Zero-shot 或简单的随机示例，AI 可能会漏掉"吃了一半"这个步骤，直接做加减法。

## 使用技巧

在日常使用 LLM（如 ChatGPT, Claude）时，你可以手动模拟 Active-Prompt 的过程：

1. **如何发现"好"的示例（计算不确定性）？**
   - **重复提问法**：开启一个新的对话窗口，把你要作为示例的问题问 3 次。如果 AI 3 次回答的逻辑或答案不一致，这个问题就是绝佳的 Prompt 素材。
   - **Self-Consistency 检查**：让 AI 生成多个思维链路径，如果路径分歧很大，说明该问题难度适中，适合作为教学案例。

2. **多样性原则 (Diversity)**：
   - 不要只选同一种类型的"难题"。如果选出的不确定性问题都关于"加减法"，那么模型可能学不会"逻辑推理"。
   - 应从不同类别的问题中各选一个高不确定性的代表。

3. **人工注解的重要性**：
   - Active-Prompt 的上限取决于**人工编写的 CoT 质量**。
   - 在编写示例答案时，要极其详细，把隐含的常识都写出来（如上面的"10的一半是5"），这样模型才能学会其中的逻辑跃迁。

4. **动态更新**：
   - 如果你发现模型在某一类新问题上表现不好，就将那个新问题加入到你的 Prompt 示例库中（重走一遍 Active-Prompt 流程）。

## 总结对比

| 方法 | 示例选择方式 | 优点 | 缺点 |
|------|-------------|------|------|
| **Zero-Shot** | 无示例 | 简单、省钱 | 复杂推理能力差 |
| **Manual CoT (传统)** | 人工随机挑选 | 效果比 Zero-shot 好 | 可能挑选到简单无效的例子 |
| **Auto-CoT** | 聚类后自动挑选 | 自动化程度高 | 示例可能有错，质量不稳定 |
| **Active-Prompt** | **挑选模型"最不确定"的题目** | **效果通常最佳，针对性强** | 需要更多计算（测不确定性）和人工介入 |

## 不确定性度量方法详解

| 方法 | 描述 | 适用场景 |
|------|------|---------|
| **不一致性 (Disagreement)** | 多次采样，统计答案分歧程度 | 有明确答案的问题 |
| **熵 (Entropy)** | 计算答案分布的熵值 | 多分类问题 |
| **置信度 (Confidence)** | 分析模型输出的概率分布 | 模型支持输出概率时 |
| **Self-Evaluation** | 让模型自我评估答案确定性 | 通用场景 |

## 实施建议

### 手动实施 Active-Prompt 的步骤

1. **准备问题池**：收集一批代表性的测试问题
2. **多次采样**：对每个问题运行模型 3-5 次
3. **计算不确定性**：统计答案不一致或推理路径差异
4. **排序筛选**：选择不确定性最高的 top-k 问题
5. **人工编写 CoT**：针对这些问题编写详细的思维链
6. **构建最终 Prompt**：将示例组合进 Few-shot Prompt

### 成本效益分析

| 成本项 | 说明 | 优化建议 |
|--------|------|---------|
| **计算成本** | 需要多次调用模型测不确定性 | 限制问题池大小，使用轻量级模型初筛 |
| **人工成本** | 需要专家编写高质量 CoT | 优先处理高不确定性样本，避免重复劳动 |
| **维护成本** | 新类型问题需要更新示例库 | 建立示例版本管理机制 |

> **实践建议**：Active-Prompt 是提升 Few-shot 效果的有效方法，特别适合对准确率要求高且有人力资源编写高质量示例的场景。虽然前期投入较高，但一旦建立高质量的示例库，后续复用价值巨大。

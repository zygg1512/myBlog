# ReAct (推理与行动)

ReAct (Reasoning and Acting) 是一种将大语言模型的推理（Reasoning）和行动（Acting）结合起来的提示工程范式。它让模型不仅能思考如何解决问题，还能与外部环境（如搜索引擎、数据库、API 等）进行交互，获取信息后再继续推理，直到得出最终答案。

## 概念

传统的语言模型在处理复杂任务时，要么只进行内部推理（如 Chain of Thought，只动脑不动手），要么只输出动作指令（只动手不动脑）。

而 **ReAct** 将这两者交织在一起：

- **推理 (Thought)**：模型分析当前情况，规划下一步，或者对观察到的结果进行总结。
- **行动 (Action)**：模型决定调用外部工具（如搜索、计算器、执行代码等）来获取更多信息。
- **观察 (Observation)**：外部工具执行后返回的真实世界结果，作为模型下一步推理的依据。

这种 `Thought -> Action -> Observation` 的循环正是当下各种 AI Agent（智能体）的核心运作机制。

## 应用场景

ReAct 范式极大地拓展了大模型的能力边界，使其从单纯的"文本生成器"变成了能够解决实际问题的"智能体 (Agent)"。常见的应用场景包括：

1. **智能客服与企业知识库问答**：面对用户的复杂客诉，模型可以自主决定调用订单查询 API 获取购买记录，再调用物流 API 查询进度，最后综合所有信息给用户一个准确的答复，而非单纯依赖训练数据。

2. **个人数字助理 (Personal Assistants)**：在协助用户规划行程时，模型可以先调用天气 API 查目的地的天气，再调用航班和酒店预订 API 查价格，最后通过推理生成一份结合了天气、预算和时间的完美行程单。

3. **自动化数据分析与编程助手 (如 AutoGPT, Devin)**：给出一个数据分析需求，模型可以先编写一段 Python 数据处理代码（Thought + Action），然后在沙盒中运行。如果遇到报错（Observation），它会阅读错误信息并反思，自动修改代码再次运行，直到成功输出图表和结论。

4. **自主网络研究员**：给定一个宽泛的研究主题，ReAct Agent 可以自主调用搜索引擎，阅读网页内容，发现新的相关链接并继续点击深入，最终提炼整理出一份深度的研究报告。

## 原理

**为什么能提升效果？**

1. **解决"幻觉"与知识盲区 (Grounding)**：大模型仅依靠内部参数回答问题时，容易产生幻觉或受限于知识截止日期。ReAct 允许模型通过 `Action` 获取外部最新、最准确的信息（`Observation`），从而极大地提高了回答的事实准确性。

2. **推理与行动的协同增强 (Synergy)**：推理帮助模型决定当前最优的行动（例如："我现在需要搜索 X 的年龄"），而行动获取的增量信息又为下一步推理提供了坚实的前提（例如："既然 X 是 45 岁，那么..."），两者形成正向循环。

3. **更高的可解释性与自我纠错能力**：人类可以清晰地看到模型的整个思考轨迹和工具调用过程。如果中途某个工具调用失败或返回了无用信息，模型可以通过 `Thought` 意识到该错误，并尝试新的策略（纠错能力）。

## 举例

在实际应用中，ReAct 通常需要给模型设定一个明确的循环格式规范以及可用的工具列表。

假设我们要让 AI 回答一个它内部知识库可能不知道、且需要多步搜集信息的实时问题。

**你的输入 (Prompt)：**

```
尽可能准确地回答以下问题。你可以使用以下工具：
- Search: 搜索引擎，用于查找互联网上的事实和最新信息。

请严格按照以下格式进行回复：
Question: 需要你回答的原始问题
Thought: 思考过程
Action: 需要采取的行动
Action Input: 行动参数
Observation: 行动结果
...
Final Answer: 最终答案

Question: Apple 现任 CEO 毕业于哪所大学？
```

**AI 输出：**

```
Thought: 我需要先查明 Apple 的现任 CEO 是谁，然后再查询他毕业的大学。
Action: Search
Action Input: Apple 现任 CEO
Observation: 蒂姆·库克 (Tim Cook) 是 Apple 的现任 CEO。
Thought: 现在我知道 Apple 的 CEO 是蒂姆·库克了。接下来我需要找出蒂姆·库克毕业于哪所大学。
Action: Search
Action Input: 蒂姆·库克 毕业大学
Observation: 蒂姆·库克获得了奥本大学 (Auburn University) 的工业工程学士学位，以及杜克大学 (Duke University) 的工商管理硕士学位。
Thought: 我现在已经找到了他毕业的大学信息，可以给出最终答案了。
Final Answer: Apple 现任 CEO 蒂姆·库克 (Tim Cook) 毕业于奥本大学（获得本科学位）和杜克大学（获得硕士学位）。
```

> **注意**：在真实的编程环境中，`Observation` 的内容通常不是 AI 自己编造的，而是由你的程序在解析出 `Action` 后，调用真实的搜索引擎 API，并将结果拼接到 Prompt 中再传给 AI 继续生成的。

## 实现架构

```

├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐         │
│   │ Thought  │───▶│  Action  │───▶│Observation│        │

│        ▲                                    │          │
│        │                                    │          │

│                    (循环直到得出答案)                    │
└─────────────────────────────────────────────────────────┘
```

## 工具定义规范

为了让模型知道可以使用哪些工具，需要在 Prompt 中明确定义：

```
可用工具：
1. Search[query]: 搜索引擎，query 为搜索关键词
2. Calculator[expression]: 计算器，expression 为数学表达式
3. GetWeather[location]: 获取天气，location 为城市名
4. ReadFile[path]: 读取文件，path 为文件路径

使用格式：
Action: 工具名称[参数]
```

## 与 Reflexion 的结合

ReAct 处理单轮任务，而 Reflexion 处理多轮迭代：

```
ReAct 循环失败 → Reflexion 反思原因 → 调整策略 → 新一轮 ReAct
```

这种组合可以构建出极具韧性的智能体系统。

## 常见陷阱与应对

| 陷阱 | 表现 | 应对 |
|------|------|------|
| **循环陷阱** | 模型在相同 Thought/Action 间反复 | 设置最大循环次数，引入多样性机制 |
| **工具误用** | 选择了不合适的工具或参数 | 提供详细的工具使用示例 |
| **过早终止** | 模型在信息不足时就给出答案 | 强化 Prompt 中的验证要求 |
| **幻觉观察** | 模型编造 Observation 而非等待真实结果 | 确保 Observation 由外部系统注入 |

## 代码示例框架

```python
class ReActAgent:
    def __init__(self, tools, llm):
        self.tools = tools
        self.llm = llm
        self.max_iterations = 10

    def run(self, question):
        context = f"Question: {question}\n"

        for i in range(self.max_iterations):
            # 生成 Thought 和 Action
            response = self.llm.generate(context)

            # 解析 Action
            action, action_input = self.parse_action(response)

            if action == "Final Answer":
                return action_input

            # 执行工具
            observation = self.tools[action].execute(action_input)

            # 追加到上下文
            context += f"Observation: {observation}\n"

        return "达到最大迭代次数，未能完成"
```

> **实践建议**：ReAct 是构建 AI Agent 的基础范式。成功的 ReAct 实现需要精心设计的工具集、清晰的格式规范和合理的循环控制机制。

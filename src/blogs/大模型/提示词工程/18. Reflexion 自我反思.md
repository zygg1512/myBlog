# Reflexion (反射 / 自我反思)

Reflexion 是一种赋予大语言模型"自我反思"和"自我纠错"能力的提示工程框架。

简单来说，它不再强求模型一次性给出完美答案，而是允许模型先进行尝试，然后根据外部反馈（如代码报错信息、正确率）或内部评估来**反思自己错在哪里**，并将这些反思经验作为新的提示，指导下一次尝试，直到得出正确结果。这就像人类的"试错-总结-改进"的学习过程。

## 概念

在基础的提示方法中，模型通常只有一次生成机会（One-shot）。如果生成错了，任务就宣告失败。

**Reflexion 的核心思想是建立"反思闭环"：**

1. **生成 (Actor)**：模型根据当前指令生成初步的答案、代码或行动计划。

2. **评估 (Evaluator)**：通过外部环境（如代码编译器、搜索引擎）或模型自身的打分，来评估初步结果是否正确。

3. **反思 (Self-Reflection)**：如果结果不理想，模型会被要求针对"为什么会失败"写一段反思总结，提出下一步的改进建议。

4. **重试 (Retry)**：将包含着"反思总结"的上下文再次喂给模型，让它带着经验教训重新尝试。

## 使用场景

Reflexion 极其适合那些"单次生成成功率低，但错误原因很容易被诊断"的任务：

1. **代码生成与调试 (Code Generation & Debugging)**：模型第一次写的代码可能存在语法错误或逻辑漏洞。把编译器的 Error Log 丢给模型，让它反思并修改代码，这是 Reflexion 最经典、效果最好的场景。

2. **长篇内容的逻辑校验 (Long-form Writing)**：在写完一篇长文章或推理报告后，让模型回头审视："我是否遗漏了用户的某项要求？"或者"这里的逻辑连贯吗？"，然后进行二次修改。

3. **智能体任务规划 (Agent Planning)**：当 AI Agent 在复杂环境中碰壁（比如在网页上没有找到需要的按钮，或者搜索 API 返回了空结果），它可以通过反思来调整策略（比如："刚才的搜索词太具体了，我应该换个更宽泛的关键词重试"）。

## 原理

**为什么能提升效果？**

1. **利用自然语言代替标量奖励 (Verbal Reinforcement)**：传统的强化学习只能告诉 AI "你得了 0 分"。而 Reflexion 让 AI 自己用文字写出"我得了 0 分是因为少考虑了边界条件 X，下次应该加上 `if X == 0` 的判断"。这种**自然语言反馈**具有极高的信息密度，能极其精确地指导下一步。

2. **打破"一次性通过"的瓶颈**：对于极难的任务，要求大模型一次做对是不现实的。Reflexion 允许它通过迭代来逼近正确答案，大幅降低了单次生成的难度。

3. **积累短期记忆 (Short-term Memory)**：模型生成的每一次"反思总结"都会被追加到上下文中。这意味着模型在后续的尝试中，不仅知道目标是什么，还清楚地记得"哪些坑我已经踩过了，不能再踩"。

## 举例

这里展示一个代码生成的场景，看看 Reflexion 是如何工作的。

**你的输入：**

```
[任务]：请编写一个 Python 函数求两个数的最大公约数 (GCD)。

[第一次尝试结果]：
def gcd(a, b):
    return a % b

[评估反馈]：测试用例 gcd(10, 15) 执行失败。期望输出: 5，实际输出: 10。

[指令]：请反思上述代码为什么失败，写出你的反思过程，并提供修正后的代码。
```

**AI 输出：**

```
【反思】：
我之前的代码 `return a % b` 仅仅计算了 a 除以 b 的余数，这完全不是最大公约数的定义。求最大公约数应该使用欧几里得算法（辗转相除法），即不断重复将除数和余数作为新的被除数和除数，直到余数为 0。当余数为 0 时，当前的被除数也就是最终的最大公约数。我在逻辑设计上犯了基础性错误。

【修正代码】：
def gcd(a, b):
    while b != 0:
        a, b = b, a % b
    return a
```

> **注意**：在这个例子中，AI 通过明确的"反思"步骤，找出了最初低级错误的原因，并在第二次尝试中给出了完美的解决方案。

## Reflexion 与 ReAct 的区别

由于这两者经常被一起提及，我们可以简单做个对比：

- **ReAct (推理与行动)**：它关注的是**单次尝试的内部过程**。模型边想边做（Thought-Action-Observation），比如"我觉得我需要搜一下 → 执行搜索 → 看到结果了 → 得出答案"。

- **Reflexion (自我反思)**：它关注的是**多次尝试之间的迭代**。它通常发生在一次 ReAct 流程失败之后。模型回头看："刚才那套 ReAct 流程为什么没搞定？哦，因为我搜错了词。"然后带着这个经验，再重新开启一轮全新的 ReAct 流程。

在目前最前沿的 AI Agent 设计中，往往是将两者结合：**底层用 ReAct 步步为营，外层加 Reflexion 不断试错反思**，从而打造出极具韧性的智能体。

## 评估与反思策略

| 任务类型 | 评估方式 | 反思重点 |
|---------|---------|---------|
| **代码生成** | 执行测试用例 | 错误类型、边界条件、逻辑漏洞 |
| **数学问题** | 答案比对 | 计算步骤、公式选择、单位换算 |
| **文本生成** | 人工评分/规则检查 | 遗漏要求、逻辑连贯、风格一致 |
| **任务规划** | 目标达成度 | 策略有效性、工具选择、执行顺序 |

## 实现模式

### 模式一：外部反馈驱动

```python
def reflexion_with_external_feedback(task, max_attempts=3):
    context = f"任务: {task}\n"

    for attempt in range(max_attempts):
        # 生成尝试
        result = llm.generate(context + "请生成解决方案:")

        # 外部评估（如代码执行）
        feedback = external_evaluate(result)

        if feedback.success:
            return result

        # 生成反思
        reflection = llm.generate(f"""
        尝试失败：{feedback.error_message}
        请反思失败原因并提出改进建议。
        """)

        # 更新上下文
        context += f"\n尝试 {attempt+1} 失败。反思: {reflection}\n"

    return "达到最大尝试次数"
```

### 模式二：自我评估驱动

```python
def reflexion_with_self_eval(task, max_attempts=3):
    context = f"任务: {task}\n"

    for attempt in range(max_attempts):
        # 生成并自我评估
        response = llm.generate(context + """
        请生成解决方案，并自我评估（1-10分）。
        如果评分 < 8，请说明改进方向。
        """)

        result, score, reflection = parse_response(response)

        if score >= 8:
            return result

        context += f"\n尝试 {attempt+1} 评分 {score}。反思: {reflection}\n"

    return result  # 返回最好的一次
```

## 关键成功因素

1. **明确的评估标准**：让模型知道什么是"对"的
2. **具体的反思引导**：不要只问"为什么错了"，要问"哪一步错了，怎么改"
3. **上下文管理**：控制历史长度，避免 token 超限
4. **退出机制**：设置最大迭代次数，防止无限循环

## 局限性与应对

| 局限性 | 表现 | 应对 |
|--------|------|------|
| **反思无效** | 模型反复犯同样错误 | 强制要求列出具体改进点 |
| **过度迭代** | 成功版本被后续迭代破坏 | 保留历史最佳结果 |
| **上下文膨胀** | 反思历史过长导致 token 超限 | 定期压缩历史或使用摘要 |
| **评估困难** | 无法自动判断对错 | 引入人工检查点 |

## Reflexion + ReAct 组合架构

```
外层：Reflexion 循环
├── 评估 ReAct 执行结果
├── 生成反思并调整策略
└── 启动新一轮 ReAct

内层：ReAct 循环
├── Thought: 分析当前状态
├── Action: 调用工具
└── Observation: 获取反馈
```

> **实践建议**：Reflexion 是提升系统韧性的关键技术。在代码生成、复杂推理等场景中，它能显著提升成功率。建议与 ReAct 结合使用，构建完整的 Agent 解决方案。
